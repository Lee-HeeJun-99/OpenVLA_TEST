## 3. Waypoint 기록

Hand-guiding으로 로봇을 움직이며 현재 TCP pose를 저장한다.

```bash
cd ~/robot_ws
source install/setup.bash

ros2 run vla_data_collector waypoint_recorder \
  --ros-args \
  --params-file src/OpenVLA_TEST/vla_data_collector/config/collector.yaml
```

직접 파라미터를 지정해서 실행:

```bash
ros2 run vla_data_collector waypoint_recorder \
  --ros-args \
  -p robot_id:=dsr01 \
  -p robot_model:=a0509 \
  -p output_path:=waypoints/cube_pick.json
```

저장 결과:

```text
~/robot_ws/waypoints/cube_pick.json
```

주의:

- waypoint 값은 로봇 제어용 원본 단위다.
- 위치는 `mm`, 회전은 `degree`다.
- 이 값을 `m/radian`으로 바꾸면 로봇 이동 명령이 틀어진다.

## 4. Episode 수집

저장된 waypoint를 따라 로봇이 이동하면서 ZED image, TCP pose, gripper 상태를 저장한다.

```bash
cd ~/robot_ws
source install/setup.bash

ros2 run vla_data_collector dataset_recorder \
  --ros-args \
  --params-file src/OpenVLA_TEST/vla_data_collector/config/collector.yaml \
  -p episode_id:=episode_000001
```

다음 episode 예:

```bash
ros2 run vla_data_collector dataset_recorder \
  --ros-args \
  --params-file src/OpenVLA_TEST/vla_data_collector/config/collector.yaml \
  -p episode_id:=episode_000002
```

저장 결과:

```text
~/robot_ws/raw_dataset/episode_000001/
├── metadata.json
├── steps.jsonl
└── images/
```

주의:

- 같은 `episode_id`가 이미 존재하면 실행이 실패한다.
- 새 수집은 `episode_000002`, `episode_000003`처럼 새 ID를 사용한다.
- 저장되는 `tcp_pose`는 RLDS/OpenVLA용 `m/radian`으로 변환된다.

## 5. 7차원 Action 생성

`steps.jsonl`에서 연속 pose 차이를 계산해 7D action을 만든다.

```bash
cd ~/robot_ws
source install/setup.bash

ros2 run vla_data_collector make_actions raw_dataset/episode_000001
```

생성 결과:

```text
~/robot_ws/raw_dataset/episode_000001/steps_with_actions.jsonl
```

Action 형식:

```text
[dx, dy, dz, droll, dpitch, dyaw, gripper]
```

## 8. RLDS/TFDS 변환

`tfds build` 대신 전용 스크립트를 사용한다.

```bash
conda activate openvla

cd ~/robot_ws/src/OpenVLA_TEST/rlds/doosan_a0509
export PYTHONPATH=~/robot_ws/src/OpenVLA_TEST/vla_data_collector:$PYTHONPATH

python build_dataset.py \
  --raw-dataset-dir ~/robot_ws/raw_dataset \
  --data-dir ~/tensorflow_datasets \
  --overwrite
```

결과 저장 위치:

```text
~/tensorflow_datasets/doosan_a0509/1.0.0/
```

결과 확인:

```bash

```

정상 파일:

```text
dataset_info.json
features.json
doosan_a0509-train.tfrecord-00000-of-00001
```

실제 RLDS 데이터 파일:

```text
~/tensorflow_datasets/doosan_a0509/1.0.0/doosan_a0509-train.tfrecord-00000-of-00001
```

## 9. RLDS 데이터 로딩 확인

```bash
conda activate openvla

python -c "import tensorflow_datasets as tfds; \
b=tfds.builder('doosan_a0509', data_dir='/home/ubuntu/tensorflow_datasets'); \
ds=b.as_dataset(split='train'); \
ex=next(iter(ds)); \
step=next(iter(ex['steps'])); \
print(ex['episode_metadata']['episode_id'].numpy().decode()); \
print(ex['episode_metadata']['num_steps'].numpy()); \
print(step['observation']['image'].shape); \
print(step['observation']['state'].shape); \
print(step['action'].shape); \
print(step['language_instruction'].numpy().decode())"
```

정상 예:

```text
episode_000001
91
(720, 1280, 3)
(7,)
(7,)
pick up the cube
```
