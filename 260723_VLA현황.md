# 260723 VLA 현황 정리

작성일: 2026-07-23

## 목표

Doosan A0509 로봇과 ZED 2i 카메라를 사용해 OpenVLA 학습용 데이터를 수집하고, 수집된 episode를 RLDS/TFDS 포맷으로 변환한다.

현재 진행 상태:

- waypoint 기록 완료
- waypoint 기반 episode 수집 완료
- 7차원 action 생성 완료
- RLDS/TFDS 변환 완료
- 다음 단계: OpenVLA repo에 dataset 등록 후 fine-tuning 실행

## 파일별 역할

`waypoint_recorder.py`

- Hand-guiding으로 로봇 TCP pose를 기록한다.
- Enter를 눌렀을 때 현재 pose를 waypoint로 저장한다.
- 저장 단위는 로봇 제어용 원본 단위다.
- 위치: `mm`
- 회전: `degree`

`dataset_recorder.py`

- `waypoints/cube_pick.json`의 waypoint를 순서대로 따라간다.
- 이동 중 ZED RGB image, TCP pose, gripper 상태를 저장한다.
- 로봇 제어에는 waypoint 원본 단위 `mm/degree`를 그대로 사용한다.
- dataset 저장 시에는 RLDS/OpenVLA용으로 `m/radian`으로 변환한다.

`make_actions.py`

- `steps.jsonl`에서 연속 step 간 차이를 계산한다.
- 7차원 action을 생성한다.

```text
[dx, dy, dz, droll, dpitch, dyaw, gripper]
```

`validate_episode.py`

- episode의 metadata, step 수, image 파일, action shape를 검증한다.

`rlds_utils.py`

- raw episode를 RLDS 변환 전에 읽고 검증하는 공통 유틸이다.

`check_rlds_source.py`

- RLDS 변환 전 raw dataset 상태를 확인하는 CLI다.

`build_dataset.py`

- `raw_dataset`을 TFDS/RLDS 포맷으로 변환한다.
- `tfds build` CLI 대신 사용한다.
- `tfds build`는 `apache_beam` 의존성 문제로 실패할 수 있어서 전용 스크립트를 사용한다.

## Waypoint 데이터 형식

저장 위치:

```text
~/robot_ws/waypoints/cube_pick.json
```

형식:

```json
{
  "instruction": "pick up the cube",
  "task": "cube_pick",
  "position_unit": "millimeter",
  "rotation_unit": "degree",
  "waypoints": [
    {
      "name": "home",
      "tcp_pose": [x, y, z, rx, ry, rz],
      "gripper": 1
    }
  ]
}
```

주의:

- 이 파일은 로봇 제어 입력이다.
- `tcp_pose` 값을 `m/radian`으로 바꾸면 로봇 이동 명령이 틀어진다.

## Raw Episode 데이터 형식

저장 위치:

```text
~/robot_ws/raw_dataset/episode_000001/
```

구조:

```text
raw_dataset/episode_000001/
├── metadata.json
├── steps.jsonl
├── steps_with_actions.jsonl
└── images/
    ├── 000000.jpg
    ├── 000001.jpg
    └── ...
```

`steps.jsonl`의 주요 필드:

```json
{
  "step_index": 0,
  "timestamp": 0.0,
  "image_timestamp": 0.0,
  "image": "images/000000.jpg",
  "tcp_pose": [x, y, z, roll, pitch, yaw],
  "gripper": 1
}
```

`steps_with_actions.jsonl`의 주요 필드:

```json
{
  "step_index": 0,
  "image": "images/000000.jpg",
  "tcp_pose": [x, y, z, roll, pitch, yaw],
  "gripper": 1,
  "action": [dx, dy, dz, droll, dpitch, dyaw, gripper]
}
```

Raw episode 저장 단위:

- `tcp_pose[:3]`: meter
- `tcp_pose[3:]`: radian
- `action[:3]`: meter delta
- `action[3:6]`: radian delta
- `action[6]`: gripper target
- `gripper`: `0=closed`, `1=open`

현재 확인된 episode:

```text
episode_000001
steps: 91
success: True
image shape: 720 x 1280 x 3
action shape: 7
```

## 그리퍼 제어

기존 controller digital output 방식에서 tool digital output pulse 방식으로 변경했다.

Open:

```text
Tool DO[1] = 1
sleep 1 sec
Tool DO[1] = 0
sleep 1 sec
Tool DO[1] = 1
sleep 1 sec
Tool DO[1] = 0
sleep 1 sec
```

Close:

```text
Tool DO[2] = 1
sleep 1 sec
Tool DO[2] = 0
sleep 1 sec
```

Config:

```yaml
gripper_open_output_index: 1
gripper_close_output_index: 2
gripper_pulse_time_sec: 1.0
```

## RLDS/TFDS 변환

변환 전 확인:

```bash
conda activate openvla
cd ~/robot_ws
export PYTHONPATH=~/robot_ws/src/OpenVLA_TEST/vla_data_collector:$PYTHONPATH

python -m vla_data_collector.check_rlds_source raw_dataset
```

정상 출력 예:

```text
Convertible episodes : 1
Total steps          : 91
Dataset units        : position=meter, rotation=radian
Action shape         : (7,)
```

RLDS 변환:

```bash
conda activate openvla
cd ~/robot_ws/src/OpenVLA_TEST/rlds/doosan_a0509
export PYTHONPATH=~/robot_ws/src/OpenVLA_TEST/vla_data_collector:$PYTHONPATH

python build_dataset.py \
  --raw-dataset-dir ~/robot_ws/raw_dataset \
  --data-dir ~/tensorflow_datasets \
  --overwrite
```

변환 결과 저장 위치:

```text
~/tensorflow_datasets/doosan_a0509/1.0.0/
```

현재 생성된 파일:

```text
~/tensorflow_datasets/doosan_a0509/1.0.0/
├── dataset_info.json
├── features.json
└── doosan_a0509-train.tfrecord-00000-of-00001
```

각 파일 의미:

- `doosan_a0509-train.tfrecord-00000-of-00001`: 실제 RLDS 데이터 shard
- `dataset_info.json`: dataset 이름, version, split 정보
- `features.json`: RLDS field schema

## RLDS Field Schema

RLDS step 구조:

```text
steps/observation/image          uint8[H, W, 3], RGB
steps/observation/EEF_state      float32[6]
steps/observation/gripper_state  float32[1]
steps/observation/state          float32[7]
steps/action                     float32[7]
steps/language_instruction       string
steps/language_embedding         float32[512]
steps/is_first                   bool
steps/is_last                    bool
steps/is_terminal                bool
steps/reward                     float32
steps/discount                   float32
```

Episode metadata:

```text
episode_metadata/file_path
episode_metadata/episode_id
episode_metadata/success
episode_metadata/num_steps
```

## 현재 주의사항

- 현재 episode는 1개뿐이므로 학습 검증용으로는 가능하지만 일반화 학습에는 부족하다.
- cube 위치, 조명, 배경, 접근 경로를 바꿔 여러 성공 episode를 더 수집해야 한다.
- 실패 episode는 초기 fine-tuning에서는 제외하는 것이 안전하다.
- `raw_dataset` 원본과 `tensorflow_datasets` 변환 결과는 별도다.
- OpenVLA 학습 시 dataset root는 개별 `.tfrecord`가 아니라 `~/tensorflow_datasets`를 바라보게 해야 한다.
